# Japan Visa Dashboard

Next.js frontend for the Japan visa prediction dashboard.

The browser talks to same-origin Next API routes under `/api/*`. Those routes proxy to the FastAPI backend. Local development defaults to `http://127.0.0.1:8000`; production Cloud Run deployments set `BACKEND_BASE_URL` from the encrypted deployment environment and use the frontend service identity to call the private backend.

The proxy applies bounded in-memory caching and per-client rate limiting before it calls the backend. This is intentionally lightweight: it reduces accidental or casual abuse without adding another paid service.

The Cloud Run deploy target caps frontend scale with `MAX_INSTANCES` and `CONCURRENCY`. Keep a GCP budget alert on the project because the in-memory limiter is per instance and should not be treated as the hard cost-control boundary.

## Commands

Use the frontend Nix shell:

```bash
nix develop
make help
```

Common commands:

```bash
make deps
make dev
make check
```

## Browser Tests

The main Playwright happy paths replay minimal API HAR cassettes from `tests/e2e/hars`. Normal test runs never update cassettes and abort unmatched API requests.

Run the browser suite:

```bash
make e2e
```

Create missing cassettes from the live backend:

```bash
make e2e-har-record
```

Refresh existing cassettes from the live backend after an intentional backend contract change:

```bash
make e2e-har-update
```

Record or update one cassette:

```bash
make e2e-har-record-one CASSETTE=prediction-default
make e2e-har-update-one CASSETTE=prediction-default
```

HAR recording uses `.env.test`, generated by `make secrets`, which points at the deployed private backend and uses local `gcloud auth print-identity-token` credentials for Cloud Run authentication.

HAR recording is scoped to `/api/meta/latest` and `/api/predictions` and uses Playwright's minimal update mode. Keep synthetic edge cases as focused inline mocks instead of recording one-off backend states.

## Environment Files

Environment files are encrypted under `secrets/` and decrypted to local ignored files:

- `secrets/.env.development.enc`: local development defaults for `make dev`.
- `secrets/.env.test.enc`: Playwright and HAR recording defaults.
- `secrets/.env.production.enc`: production build, source-map upload, and deploy defaults.

Run `make secrets` to decrypt them into local `.env.development`, `.env.test`, and `.env.production` files. Root `.env` and `.env.local` are not generated. Personal overrides should be made by editing the appropriate encrypted env file or by exporting variables for a single command.

## Local Backend

Run the backend separately, then start the frontend:

```bash
BACKEND_BASE_URL=http://127.0.0.1:8000 make dev
```

If `BACKEND_BASE_URL` is omitted outside production, the proxy uses the local backend default. In Cloud Run, or when `ENVIRONMENT=production`, the proxy fails fast when `BACKEND_BASE_URL` is missing.

## CI/CD

Pushes and pull requests run frontend checks.

Deployments run only from version tags matching `v*`. The deploy workflow installs dependencies, runs checks, authenticates to Google Cloud, decrypts deployment secrets with SOPS, builds the Docker image using the tag name, pushes it, and deploys it to Cloud Run.

Release tags are managed from `VERSION`. Use `make hotfix` for a patch release and `make release` for a minor release. Both targets bump `VERSION`, sync `package.json`, update `CHANGELOG.md`, create an annotated tag, and push the branch with tags. CI runs `make version-check`; tag deploys fail if the pushed tag does not match `VERSION`.

`make service-deploy` reads `BACKEND_BASE_URL` from `secrets/.env.production.enc` and writes it to the frontend Cloud Run service alongside `ENVIRONMENT=production` and `BACKEND_AUTH_MODE=google`.

`make image-build` also runs `make secrets`, but passes a temporary decrypted production env file into Docker as a BuildKit secret for `next build`. The file is not created in the repo and is not copied into the runtime image. Production-only server values are set as Cloud Run runtime environment variables during `make service-deploy`.

Production browser source maps are generated for private Rollbar upload, not for public serving. `make image-push` builds a separate source-map artifact image, uploads JavaScript maps to Rollbar with the current Git commit as the code version, and pushes a runtime image that has `.map` files removed. Use `make runtime-sourcemaps-check` after `make image-build` to verify the runtime image remains map-free.

Use `make registry-prune` to preview old GCR image deletion. The cleanup reads `VERSION`, refuses to run if that tag is not present in the registry, keeps the image digest for that tag, and keeps the two newest digests by registry update time so the current deploy and the image it replaced remain available. After reviewing the preview, run `make registry-prune-apply` to delete the remaining digests.

## Proxy Controls

Optional runtime settings:

- `FRONTEND_PROXY_RATE_LIMIT`: requests per client per route window. Default: `60`.
- `FRONTEND_PROXY_RATE_LIMIT_WINDOW_SECONDS`: rate limit window. Default: `60`.
- `FRONTEND_PROXY_RATE_LIMIT_MAX_BUCKETS`: maximum in-memory rate limit buckets per instance before oldest buckets are pruned. Default: `10000`.
- `META_LATEST_PROXY_CACHE_SECONDS`: cache TTL for `/api/meta/latest`, in seconds. Default: `604800` / one week.
- `PREDICTIONS_PROXY_CACHE_SECONDS`: cache TTL for `/api/predictions`, in seconds. Default: `604800` / one week.
- `BACKEND_PROXY_CACHE_MAX_ENTRIES`: maximum in-memory backend response cache entries per instance. Default: `500`.
- `BACKEND_IDENTITY_TOKEN_TIMEOUT_MS`: timeout for fetching the Google identity token used to call the private backend. Default: `2000`.
- `BACKEND_PROXY_TIMEOUT_MS`: timeout for frontend proxy calls to the backend API. Default: `10000`.

The backend data updates monthly, so one-week cache defaults keep repeat queries cheap while still limiting stale responses after a fresh import.
